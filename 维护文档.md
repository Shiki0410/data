# 项目结构与维护说明（Baseball Interactive Story）

## 1. 项目概述
本项目是基于 React + TypeScript + Vite + ECharts 的棒球数据可视化网站，核心内容是山本由伸（投手）与大谷翔平（打者）的 Statcast 交互分析，并整合 Dodgers 队级数据。

## 2. 当前实际入口
- `index.html` -> 加载 `/src/main.tsx`
- `src/main.tsx` -> 渲染 `src/App.tsx`
- 主要样式：`src/Layout.css`、`src/Typography.css`

> 根目录的 `App.tsx` / `main.tsx` 为旧版本文件，当前主站不走这条链路。

## 4. 技术栈
- React 18
- TypeScript 5
- Vite 5
- ECharts 5
- GSAP + ScrollTrigger
- Lenis
- d3-dsv（CSV 解析）
- Three.js（WebGL 视觉模块）

## 5. 数据加载与处理
`src/App.tsx` 使用 `Promise.all` 读取 4 个核心 CSV：
1. 山本投球逐球数据
2. 大谷击球逐球数据
3. Dodgers 打者汇总
4. Dodgers 投手汇总

常见处理：
- `toNum()` 数值转换
- `Number.isFinite()` 清洗无效值
- PA 阈值过滤（避免样本过小）

## 6. 页面章节
- `hero`：主视觉与球员切换
- `fragments`：叙事碎片卡
- `macro`：宏观指标与速度演出
- `lab`：核心可视化实验区
- `outro`：结尾与回放

## 7. 已实现图表（lab 区）
- Dodgers Bat Top10（wOBA）
- Dodgers Pitch K% vs BB%
- 打者能力雷达
- 投手能力雷达
- 能力图总览（多维表格 + 分位条）
- Pitch Location
- Pitch Movement
- Release Points
- Count Breakdown
- EV vs LA
- Spray Chart
- Event Breakdown

## 8. 关键交互
- 打者筛选（默认 Ohtani）
- 投手筛选（默认 Yamamoto）
- Pitching / Batting 模块菜单按钮（Savant 风格入口）
- 下拉框高对比颜色：打者蓝色系、投手金色系
- 球员头像联动：`player_id -> MLB headshot URL`

## 9. 重要函数/状态（`src/App.tsx`）
- 工具函数：
  - `toNum(v)`：转数字
  - `shortName(name)`：短名
  - `percentileRank(arr, value, lowIsBetter?)`：分位计算
  - `headshotUrl(playerId)`：头像 URL 生成
- 核心状态：
  - `selectedBatter`
  - `selectedPitcher`
  - `selectedPitchModule`
  - `selectedBatModule`
  - `pitchRows / batRows / teamBatRows / teamPitchRows`

## 10. 样式维护重点（`src/Layout.css`）
重点类名：
- `.chart-grid`, `.chart-card`, `.chart-box`
- `.panel-filter-row`, `.player-select`, `.player-chip`
- `.ability-matrix*`, `.pct-cell*`
- `.module-menu*`, `.module-btn`

建议：
1. 新功能样式继续往 `Layout.css` 的模块分区追加。
2. 统一复用已有颜色（打者蓝 / 投手金）。
3. 改动后必须看移动端断点（`max-width: 900px`）。

## 13. 快速排错
- 页面空白：检查 `index.html` 是否仍指向 `/src/main.tsx`
- 图表不显示：检查 ref 是否存在、CSV 是否成功加载
- 选择无变化：检查筛选值是否与 `player_name` 完全一致
- 头像不显示：检查 `player_id` 是否有效数值
