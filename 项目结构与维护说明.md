# 项目结构与维护说明（Baseball Interactive Story）

> 目的：记录当前项目的完整结构、关键模块、数据流与维护要点，便于后续迭代与协作。

---

## 1. 项目定位

这是一个基于 `React + TypeScript + Vite + ECharts` 的棒球数据可视化网站，核心围绕：

- 山本由伸（投手）与大谷翔平（打者）逐球数据可视化
- Dodgers 队级打者/投手能力对比
- Savant 风格的图表模块与交互筛选
- 叙事化滚动页面（intro / hero / fragments / macro / lab / outro）

## 4. 入口与渲染链路

1. `index.html` 加载 `/src/main.tsx`
2. `src/main.tsx` 渲染 `<App />`
3. `src/App.tsx` 为当前主站页面
4. `src/Layout.css` + `src/Typography.css` 提供主视觉

> 注意：根目录也有旧版 `App.tsx/main.tsx`，当前不会被 `index.html` 直接引用。

---

## 5. `src/App.tsx` 功能分层

## 5.1 数据类型

- `PitchRow`：投球逐球字段（plate_x/z, pfx_x/z, spin, count 等）
- `BatRow`：击球逐球字段（launch_speed/angle, spray 坐标等）
- `TeamBatRow`：队级打者汇总
- `TeamPitchRow`：队级投手汇总

## 5.2 数据加载

`useEffect + Promise.all(fetch csv)` 读取四个核心 CSV：

- 山本投球
- 大谷击球
- Dodgers 打者汇总
- Dodgers 投手汇总

并在加载后做数值清洗：`toNum`、`Number.isFinite`、PA 门槛过滤。

## 5.3 页面章节

- `hero`：视觉开场（可切 Yamamoto / Ohtani 底图）
- `fragments`：横向叙事卡片
- `macro`：宏观指标与速度数字演出
- `lab`：完整图表面板（本项目核心）
- `outro`：收尾与重播

## 5.4 核心图表（ECharts）

- Dodgers Bat Top10 (wOBA)（条形图）
- Dodgers Pitch K% vs BB%（散点图）
- 打者能力 Radar
- 投手能力 Radar
- Pitch Location
- Pitch Movement（按球种分色）
- Release Point
- Count Breakdown（热力图）
- EV vs LA
- Spray Chart
- Event Breakdown（环形图）

## 5.5 新增交互（近期）

- 打者/投手能力总览：多维能力矩阵（分位条）
- Pitching / Batting Modules 菜单按钮（Savant 风格入口）
- 打者与投手下拉框强化配色（可视性提升）
- 球员头像联动：`headshotUrl(player_id)` 动态加载 MLB 头像

## 6. 样式维护说明（`src/Layout.css`）

当前样式集中在 `Layout.css`，按模块维护：

- 基础层：`.aw-root`, `.section`, `.chart-grid`, `.chart-card`
- 叙事层：intro/hero/fragment/macro/outro
- 图表层：`.chart-box`
- 交互层：`.panel-filter-row`, `.player-select`, `.module-btn`
- 能力矩阵：`.ability-matrix*`, `.pct-cell*`

维护建议：

1. 新增样式优先按“功能块”分段，不要散落。
2. 移动端规则继续收敛在现有 `@media (max-width: 900px)` 内。
3. 颜色变量可在后续统一抽到 `:root` 变量，减少硬编码。

## 7. 数据与字段约定

## 7.1 关键字段

- 投球：`pitch_name`, `release_speed`, `plate_x`, `plate_z`, `pfx_x`, `pfx_z`, `release_pos_x/z`, `balls`, `strikes`
- 击球：`launch_speed`, `launch_angle`, `hc_x`, `hc_y`, `events`
- 汇总：`woba`, `k_percent`, `bb_percent`, `hardhit_percent`, `barrels_per_pa_percent`, `velocity`, `hrs`, `pa`

## 7.2 默认联动逻辑

- 投球详情默认只在 `selectedPitcher === Yamamoto` 时使用山本逐球数据
- 击球详情默认只在 `selectedBatter === Ohtani` 时使用大谷逐球数据

> 若未来支持更多球员逐球细节，需要新增更多球员逐球 CSV，并扩展 `activePitchRows/activeBatRows` 的映射逻辑。

## 8. 外部资源依赖

## 9. 维护清单（每次改动建议）

1. 修改 `src/App.tsx` 后，检查是否有重复 `useMemo/useEffect` 与依赖数组问题。
2. 新增图表时，确认：
   - `ref` 唯一
   - `echarts.init` 与 `dispose` 成对
   - resize 监听已清理
3. 改样式后检查移动端断点（<=900px）。
4. 数据 CSV 更新后，确认字段名未变化（大小写、下划线）。
5. 跑一次构建与 lint（至少检查改动文件）。
